JavaScript: Classes

Important: Access to instance members of a class always with this. 


class Person {
  constructor(name) {
      this._name = name;
  }
  
  get name() {
      return this._name;
  }
  
  set name(newName) {
      this._name = newName;
  }
  
  toString() { return "Person: " + this.name + ""; }
}
            
const p = new Person("Susi");
p.toString() // Person: Susi
          

p.name // Susi (getter)
p.name = "Lisa"; // setter
console.log(p.toString()) // Person: Lisa;
            

p._name // Lisa (direct field access, no visibility restrictions!)  üò±
p._name = "Moni"; // (direct field access, no visibility restrictions!) üò±
console.log(p.toString()) // Person: Moni;

JavaScript: Classes #2


class Musician extends Person {
  constructor(name, instrument) {
    // call to 'super' always must be first expression!
    super(name);
    this._instrument = instrument;
  }

  toString() { return super.toString() + " plays " + this._instrument}
}            

const lemmy = new Musician("Lemmy", "Bass");

Modules

Modules

Default exports

Objects, functions and classes needs to be exported and imported explicitly

Adding an export to a file makes it a module!


            // src/domain/Person.js
            export default class Person { ... }

            or

            class Person { ... }
            export default Person
          

              // src/domain/Person.js
              export default function createPerson() { ... }
  
              or
  
              function createPerson() { ... }
              export default createPerson;
            

                Only ONE default export possible:
                export default function createPerson() { ... }
                export default class Person { ... } // KONFLIKT

Modules

Imports

When importing a module, you have to specify the Path(!) to it (in Browser even with .js-suffix!) 


              // src/domain/Musician.js
              import Person from "./Person.js";
  
              export default class Musician extends Person { ... }
            

                // src/App.js
                import Musician from "./domain/Musician.js";
  
                const lemmy = new Musician("Lemmy", "Bass");

Modules

Named exports

Beside the (one) default export, a module can contain one or more named exports 


                // src/domain/Person.js
                export default class Person { ...}
                export function createPerson() { ... }
                export function removePerson() { ... }
              

                  // src/domain/Person.js
                  export default class Person { ...}
                  function createPerson() { ... }
                  function removePerson() { ... }

                  export { createPerson, removePerson }
                

                  // src/domain/Person.js
                  export default class Person { ...}
                  function createPerson() { ... }
                  function removePerson() { ... }

                  export { createPerson as newPerson, removePerson }

Modules

Named imports


                    // src/App.js
                    import { createPerson } from "./Person.js"
              

                // Import default and named imports together
                import Person, { createPerson } from "./Person.js"
              

                  // Import everything
                  import { * as P } from "./Person.js"
  
                  P.createPerson();
                

                    // Rename in import (for example to avoid naming clashes)
                    import { createPerson as create } from "./Person.js"
    
                    create();

Modules: Resolving process

The specified path is resolved differently in node and in the browser

As of today you only have to know the Node resolving process (Webpack takes care of the rest)

Relative path is your own module

"Absolute" path (without leading ./) is searched under node_modules

Modules in the browser

To use modules in the browser (inside a script-element) you need to set the script type to module

Not (yet) relevant when developing apps using React (or Angular) as modules are "removed" during build 


            <html>
              ...
              <script type="module">
                import Musician from "./Musician.js";

                const m = new Musician("Lemmy", "Bass");
              </script>
            </html>

Toolstack

Typical tools used when building SPAs

  • NodeJS: used to run tools outside the browser
  • npm: Node Package Manager for managing external dependencies
  • yarn: One of several npm alternatives, very popular in React community
  • babel (or typescript): A compiler (sometimes also called "transpiler") that translates your modern javascript to back into a JavaScript version all browser understand
  • webpack: A bundler that transforms your modularized source code into code that is understood by all browsers
  • ESlint: ensures static code quality (like spaces vs tabs ü•≥)
  • Prettier: popular code formatter
